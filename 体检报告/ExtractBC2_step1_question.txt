核心任务： 你的任务是充当一个健康数据分析师。你需要处理一个或多个JSON 格式的年度体检报告，对它们进行历年对比分析，然后生成一个单独的 JSON 汇总对象，该对象必须严格遵循 体检比较JsonSchema.JSON 的结构。

输入文件：

多个体检报告 JSON 文件 (例如: 2022.json, 2023.json, 2024.json ...)。这些是你的唯一数据源。

注意： 文件的数量和年份是不固定的。你必须处理所有上传的此类文件。

一个 体检比较JsonSchema.JSON 文件。这是你输出 JSON 必须严格遵守的结构。

执行步骤：

加载与排序：

加载所有提供的体检报告 JSON 文件。

分析每个文件中的 patientInfo.examDate，并按时间从早到晚对报告进行排序。

填充 headerInfo：

从最新的（排序后的最后一个）报告中，提取 patientInfo.name, patientInfo.gender, 和 patientInfo.age (作为 latestAge)。

根据最早和最晚的报告年份，生成 yearRange 字符串 (例如："2022-2024")。

填充 keyVitals：

exams 数组：为每个报告（按时间顺序）创建一个对象，包含其 examDate 和 age。

metrics 数组：识别所有报告中共有的关键体征（例如 weight, bmi, bloodPressure_Systolic / bloodPressure_Diastolic）。为每个体征创建一个对象，并按时间顺序从每个报告中提取相应的值，填充 values 数组。

示例：{ "name": "体重 (kg)", "values": [58.0, 59.0, 60.5] }

生成 chartData (分析)：

labels: 创建一个包含所有已排序报告年份的数组 (例如: ["2022", "2023", "2024"])。

分析所有 labResults，选择最有趋势性或最关键的数值型指标（例如 空腹葡萄糖, 总胆固醇, 脂蛋白 a, 血尿酸）以及 patientInfo 中的 weight 和 bmi。

从每一个 JSON 报告中按顺序提取这些指标的数值，填充到 bmiChart, lipidChart, uricAcidChart 下的对应数组中。

注意：如果某年的数据为 null 或不存在，应在数组的相应位置使用 null。

生成 statusSummary (分析)：

improvements (改善项): 通过跨年比较，找出明确的积极变化。

例如：labResults 中某指标 isAbnormal 从 true 变为 false（如 2023.json 的 尿蛋白 为 ++，但在 2024.json 中为 (-)）。

例如：某个异常值在持续向参考范围靠拢（如 血尿酸 逐年下降）。

将这些发现总结为字符串数组。

risks (风险项): 找出持续存在的问题或负面趋势。

例如：labResults 中某指标在所有年份中 isAbnormal 均为 true（如 总胆固醇 或 脂蛋白 a）。

例如：imagingResults 中持续存在的发现（如 "右肺上叶微小结节，与去年对比未见明显变化"）。

例如：patientInfo.bmi 逐年升高并进入“超重”范围。

将这些发现总结为字符串数组。

生成 aiSummary (分析)：

这是基于你上述所有分析的综合书面总结。

title: 创建一个合适的标题 (例如: "综合评价 (2022-2024)")。

introduction: 撰写一段引导性文字，总结客户的整体健康状况和最明显的变化趋势。

points: 基于 statusSummary 的分析，撰写3-5个关键点，清晰地分为改善的方面、保持稳定的方面和需要重点关注的风险。

conclusion: 根据“风险项”分析，提供 1-2 句最核心的健康建议。

输出要求：

不要包含任何解释性文字、问候或对话。

仅在一个 JSON 代码块中提供最终生成的、严格符合 体检比较JsonSchema.JSON 的 JSON 对象。
